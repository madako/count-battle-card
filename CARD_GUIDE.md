# カード追加ガイド

このゲームのカードはすべて `game.js` の先頭にある `CARD_TYPES` という配列(リスト)に
1つずつ書かれています。新しいカードを増やしたい時は、**この配列にオブジェクトを
1つ追加するだけ**です。ビルドやコンパイルは不要で、保存してブラウザを再読み込みすれば
すぐ反映されます。

このガイドは「よくあるパターン」をコピペで使えるテンプレートとして用意したものです。
まずは自分が作りたいカードに近いパターンを探して、`desc`(説明文)や数値、色を書き換える
だけで大抵のカードは作れます。

## 手順

1. 下の「パターン集」から近いものを探す
2. `CARD_TYPES` 配列の最後(`];` の直前)にテンプレートをコピペする
3. `id` をユニークな名前に変える(既存のidと絶対に被らないように。一覧は
   `assets/cards/README.md` にあります)
4. `name` / `desc` / `color` を実際のカード内容に書き換える
5. ブラウザで `index.html?debug` を開いて動作確認する(下記「動作確認ツール」参照)
6. うまく動いたらイラストを用意する(無くても色背景で普通に遊べます。詳しくは
   `assets/cards/README.md`)

`id` は英数字とアンダースコア(`_`)のみを使ってください(例: `double_limit`)。
日本語や記号は使わないでください。

## パターン集

### A. 一番シンプルなカード(即座に効果が発動して終わり)

「上限を2倍にする」「進行方向を逆にする」のようなカード。まずはこれが基本形です。

```js
{
  id: "sample_simple",                 // ← ユニークなid
  name: "サンプルカード",               // ← 手札などに表示される名前
  desc: "使うと何かが起きる",           // ← 効果の説明文(ルール一覧にも表示される)
  color: "#3498db",                    // ← カードの色(下の「色の選び方」参照)
  effect(state, playerIndex) {
    // ここに実際の処理を書く。例えば上限を+10するなら:
    // state.limit += 10;
    return "何かが起きた!";           // ← ログに表示される文言を返す
  },
},
```

### B. 「このターンだけ最大値+N」タイプ

`count_boost`(カウントブースト)や `dash`(ダッシュ)と同じ形です。

```js
{
  id: "sample_boost",
  name: "サンプル(ブースト)",
  desc: "このターンだけ、進められる数の最大値が+3される",
  color: "#e67e22",
  magnitude: 3,                        // ← +3の「3」の部分。up gradeなどのencで倍率がかかる
  effect(state, playerIndex, context) {
    const mag = context.magnitude != null ? context.magnitude : 3;
    state.turnAdvanceBonus += mag;
    return `このターンの最大カウント数が+${mag}された!`;
  },
},
```

### C. 何度でも使える「速攻」カードにしたい場合

上のどのパターンにも `sub: ["速攻"]` を1行足すだけです(1ターン1枚の制限を無視できる)。

```js
{
  id: "sample_speedy",
  name: "サンプル(速攻)",
  desc: "(速攻)何かが起きる。何枚でも重ねて使える",
  color: "#f39c12",
  sub: ["速攻"],                        // ← これを足すだけ
  effect(state, playerIndex) {
    return "何かが起きた!";
  },
},
```

### D. 対象を1人選ぶ「攻撃」カード

`steal`(steal)と同じ形です。`needsTarget: true` を付けると、使用時に対象プレイヤーを
選ぶ画面が自動で出ます。選ばれた相手は `context.targetIndex` に入っています。

```js
{
  id: "sample_attack",
  name: "サンプル(攻撃)",
  desc: "(攻撃)対象を1人選び、不利な効果を与える",
  color: "#c0392b",
  category: "attack",                  // ← reflectionカードで跳ね返せるようにするために必要
  needsTarget: true,
  effect(state, playerIndex, context) {
    const target = state.players[context.targetIndex];
    // ここで target に対して何かする。例えば手札を1枚捨てさせるなら:
    // if (target.hand.length) target.hand.pop();
    return `${target.name}に効果を与えた!`;
  },
},
```

### E. 使う時に選択肢(A or B)を選ばせたいカード

`time_trick`(タイムトリック、+5か-5か選ぶ)と同じ形です。選ばれた選択肢は
`context.choice` に、`choices` で書いた `key` の値が入ります。

```js
{
  id: "sample_choice",
  name: "サンプル(選択式)",
  desc: "AかBを選んで発動する",
  color: "#2ecc71",
  choices: [
    { key: "a", label: "Aを選ぶ" },
    { key: "b", label: "Bを選ぶ" },
  ],
  effect(state, playerIndex, context) {
    if (context.choice === "a") {
      return "Aが発動した!";
    }
    return "Bが発動した!";
  },
},
```

### F. 山札を見てから1枚選んで引くカード

`choose`(choose)や `steal` のような、山札やランダム要素・enc(添付)・cha(割り込み)を
使うカードは、A〜Eより一段複雑です。無理に自力で書かず、`game.js` の中の同じ系統の
カード(`choose` は `peek`、`up_grade`/`speed_up` は `enc`、`cha_deny`/`reflection` は
`cha` の実装例)をコピーして書き換えるのが安全です。

**このパターンだけは、Claudeに「〇〇みたいな効果のカードを追加して」と頼むのが一番早くて
確実です。** effect自体は数行でも、山札操作やタイミングの扱いを間違えると他のカードとの
組み合わせでバグりやすい部分なので、ここは無理せず頼ってください。

## 色の選び方

`color` はカードのテーマカラー(16進数)です。既存カードとかぶっても問題ありません。
迷ったら、カードの雰囲気に近い既存カードの色を参考にしてください(一覧は
`assets/cards/README.md`)。ダークテーマの背景に白文字が乗るので、あまり明るすぎない
彩度高めの色(例: `#3498db`, `#8e44ad`, `#c0392b`)が馴染みます。

## 動作確認ツール(?debug)

新しいカードは山札からランダムに配られるため、狙って手札に来るとは限りません。
`index.html?debug`(ローカルで確認するなら `http://localhost:8000/?debug` のように
URLの末尾に `?debug` を付ける)でゲームを開くと、ゲーム画面のログの下に
「🛠 動作確認ツール」という欄が出てきます。プルダウンから追加したいカードを選んで
「手札に追加する」を押せば、今の手番の手札に即座に入るので、狙った通りに動くか
すぐ試せます。`?debug` を付けなければ今まで通り何も表示されません(本番プレイ時は
このURLを使わなければ問題ありません)。

## 追加後のチェックリスト

- [ ] `id` が他のカードと重複していない
- [ ] `?debug` で実際に手札に入れて使ってみて、意図した効果とログメッセージになっている
- [ ] ルール・カード一覧モーダル(「ルール・カード一覧を見る」)にも正しく表示されている
- [ ] イラストを用意する場合は `assets/cards/<id>.png`(または `.svg`)を追加し、
      `assets/cards/README.md` の一覧にも1行足しておく(後で他のカードとまとめて
      イラスト発注する時に迷わないため)

## 山札に入る枚数について

現状、カード1種類あたりの山札の枚数はセットアップ画面の「カード1種類あたりの山札枚数」
で**全カード共通**の設定になっています。特定のカードだけ多め/少なめに入れたい、といった
個別調整はまだ対応していません。必要になったらClaudeに相談してください。
