# カードイラスト配置ルール

`game.js` はカードごとに **`assets/cards/<id>.png` → `assets/cards/<id>.svg`** の順で
自動的に探して読み込みます(`buildCardVisual()`)。コード側に画像パスを1枚ずつ登録する
仕組みは無く、**ファイルを置くだけ**で反映されます。

現状、全カードに Claude が描いた簡易的な線画アイコン(`<id>.svg`)が置かれています。
PNG が優先されるので、**本物のイラストを `<id>.png` として同じフォルダに置くだけで、
コードは一切変更せずに SVG の仮絵を上書きできます**。両方とも無い/読み込みに失敗した
カードは、今まで通り色背景のみの表示にフォールバックします。

## ファイル仕様

- ファイル名: `<カードid>.png`(下記一覧の id をそのまま使う。拡張子は `.png` 固定。
  すでに `<id>.svg` がある場合は、同名の `.png` を追加するだけで自動的にそちらが
  優先表示されます。`.svg` を削除する必要はありません)
- 推奨サイズ: 260×160px 前後(横長)。表示エリアは手札で高さ64px、ルール一覧で高さ80px、
  幅いっぱいに `object-fit: cover` で切り抜き表示されるので、被写体は中央〜上寄りに収める
- 背景: カード自体に色帯があるため、イラストは背景込みで塗りつぶしでも透過でもどちらでも可
